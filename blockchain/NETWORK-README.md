# ChainCacao Network - Quick Start

## Prerequisites

- Docker & Docker Compose
- Hyperledger Fabric binaries

## Download Fabric Binaries

```bash
cd blockchain/scripts
./bootstrap-fabric.sh 2.5.0 1.5.0
```

## Complete Network Reset & Start

```bash
cd blockchain/scripts
./reset-network.sh
```

## Verify Network

```bash
./verify-network.sh
```

## Next Steps

1. Create channel: `./create-channel.sh`
2. Deploy chaincode: `./deploy-chaincode.sh`
3. Test: `./health-check.sh`

## Network Architecture

- **1 Orderer**: `orderer.chaincacao.com:7050`
- **5 Organizations** with 1 peer each:
  - OrgProducteursMSP (peer0.producteurs.chaincacao.com:7051)
  - OrgExportateursMSP (peer0.exportateurs.chaincacao.com:8051)
  - OrgCertifMSP (peer0.certif.chaincacao.com:9051)
  - OrgMinistereMSP (peer0.ministere.chaincacao.com:10051)
  - OrgTransformateursMSP (peer0.transformateurs.chaincacao.com:11051)

## Troubleshooting

- If genesis block fails: Check etcdraft configuration in configtx.yaml
- If certificates missing: Fabric CA containers must complete enrollment. Check: `docker logs ca_producteurs`
- If containers fail: Check Docker logs with `docker logs <container_name>`

## Certificate Generation

Certificates are now generated by **Fabric CA** containers instead of cryptogen:

- Each organization has its own CA container
- Identities are registered and enrolled via fabric-ca-client
- MSP structures are automatically created
- TLS certificates included in enrollment
